<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted lifecycle fixture JSON to compatibility normalizer | Duplicate approved members can change production acceptance while preserving a collapsed last value. |
| Manifest entry metadata to evidence-role authorization | Editor-controlled evidence kinds claim whether bytes are tagged-source-derived or design-derived. |
| Manifest/source values to exception and CLI diagnostics | Paths, ranges, and case identifiers may contain sensitive or control-bearing text. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-P5 | Tampering | `_normalize_tool_events`, `_normalize_terminal_fixture` | high | mitigate | Reuse production pair-preserving projectors and test every approved duplicate family plus an ignored-additive positive control. |
| T-02-P6 | Spoofing | `_normalize_lifecycle_events` evidence roles | high | mitigate | Enforce exact `_EXPECTED_KINDS` mapping for every required path in the release-agnostic path and test unknown/swapped roles. |
| T-02-P7 | Information Disclosure | `ProvenanceError`, traceback state, and `main` stderr | high | mitigate | Use closed constant codes, translate outside active exception handlers, suppress causes, and assert no canary, cause, or context across all representations. |
| T-02-P8 | Denial of Service | fixture parsing and diagnostic output | medium | mitigate | Retain existing bounded fixture inventory/SSE parsing and constrain CLI diagnostics to one closed single-line code. |
| T-02-SC | Tampering | package supply chain | low | accept | No package is added or changed; the existing locked environment is used and Phase 4 owns the current-version refresh. |
</threat_model>
