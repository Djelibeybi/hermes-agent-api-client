## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted SSE JSON to pair-aware protocol projection | Duplicate names, hostile nested objects, and coercible values cross into trusted decoding. |
| Private tool DTO to public event | Only bounded correlated facts may leave the protocol layer. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Tampering | JSON duplicate handling and `_ToolProgressWire` | high | mitigate | Preserve pairs, reject duplicates only at approved paths, require exact fields/types/bounds, and test same/conflicting duplicates as raw bytes. |
| T-02-02 | Tampering | ordered tool mapping | medium | mitigate | One-record-to-one-event mapping with repeat/interleave/interruption tests; no local reconciliation state. |
| T-02-03 | Information Disclosure | pair tree, ignored tool fields, parser failures | high | mitigate | Approved-field projection, input-independent sentinels, explicit scrub paths, and traceback/generator canary assertions. |
| T-02-DOS | Denial of Service | lifecycle identifiers | medium | mitigate | Enforce 256-character semantic caps in addition to existing 64-KiB event and 256-KiB pending-line bounds. |
| T-02-SC | Tampering | package dependencies | low | accept | Use the standard library object-pairs hook and existing Pydantic only; no install or metadata change. |
</threat_model>
