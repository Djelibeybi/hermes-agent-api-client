<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Live tag identity to detached source tree | A release name and peeled commit become the external identity used for all later evidence checks. |
| Detached source tree to local manifest entries | Editor-controlled release, commit, URL, and anchor fields must not authenticate one another. |
| Validated canonical/newer fixture bytes to compatibility decision | Public equivalence must be computed from immutable evidence rather than asserted by metadata. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-P1 | Spoofing | manifest and fixture release/commit identity | high | mitigate | Supply expected identity from independently verified tags, check source-tree HEAD, and reject manifest/entry/source-ref mismatches in adversarial tests. |
| T-02-P2 | Tampering | newer fixture bytes, hashes, and source anchors | high | mitigate | Load the newer manifest, require its lifecycle inventory, verify every hash and allowed source anchor against the newer detached commit, and test missing/stale evidence. |
| T-02-P3 | Tampering | normalized public-event equivalence | high | mitigate | Compute canonical/newer events from validated bytes and reject behavior differences even when declaration maps claim equality. |
| T-02-P4 | Denial of Service | temporary source-tree lifecycle | medium | mitigate | Reuse bounded manifest/SSE parsing and guarantee temporary checkout cleanup in all result paths. |
| T-02-SC | Tampering | package supply chain | low | accept | This plan installs no package and changes no dependency metadata; Phase 4 owns the already-recorded latest-version recheck. |
</threat_model>
